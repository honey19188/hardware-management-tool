# 单片机硬件信息管理工具 - 产品需求文档

## 1. 产品概述
单片机硬件信息管理工具是一个用于管理和追踪各类单片机硬件设备的Web应用系统。系统支持对传感器、核心板等硬件进行分类编码、生成唯一标识二维码,并记录完整的设备生命周期信息,包括硬件规格、采购信息和入库时间等。

**目标用户**: 硬件开发工程师、实验室管理人员、设备采购人员
**核心价值**: 提升硬件设备管理效率,实现设备信息的数字化追踪和快速识别

## 2. 核心功能

### 2.1 功能模块
1. **设备管理页面**: 设备列表展示、添加新设备、编辑设备信息、删除设备
2. **设备详情页面**: 设备完整信息展示、二维码生成与下载

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|---------|---------|---------|
| 设备管理页面 | 设备列表 | 展示所有设备,支持按类别、状态筛选,搜索功能 |
| 设备管理页面 | 添加设备表单 | 录入设备类别、硬件信息、采购时间、入库时间等 |
| 设备管理页面 | 批量操作 | 批量删除、批量导出数据 |
| 设备详情页面 | 基本信息展示 | 显示设备ID、类别、硬件规格、MAC地址等详细信息 |
| 设备详情页面 | 二维码模块 | 生成设备唯一标识二维码,支持下载打印 |
| 设备详情页面 | 时间线记录 | 显示设备入库、修改等时间记录 |

### 2.3 设备分类与ID编码规则

**类别前缀**:
- 传感器类: `SENSOR-`
- 核心板类: `BOARD-`
- 通信模块类: `COMM-`
- 电源模块类: `POWER-`
- 存储模块类: `STORAGE-`
- 其他: `OTHER-`

**ID格式**: `{类别前缀}{YYYYMMDD}{序号}`
示例: `SENSOR-20260625-001`

### 2.4 设备信息字段

**必填字段**:
- 设备名称
- 设备类别
- 购买时间
- 入库时间

**可选字段**:
- 核心型号(如STM32F103、ESP32等)
- MAC地址
- 固件版本
- 存储容量
- 接口类型(UART、SPI、I2C等)
- 工作电压
- 功耗参数
- 封装形式
- 生产厂家
- 备注

## 3. 核心流程

### 3.1 设备添加流程
用户点击"添加设备" → 填写设备基本信息(类别、名称、购买时间、入库时间) → 填写硬件规格(核心型号、MAC地址等) → 系统自动生成设备ID → 保存到数据库 → 自动生成二维码

### 3.2 设备查询流程
用户进入设备管理页面 → 通过搜索框或类别筛选器查找设备 → 点击设备查看详情 → 查看完整信息和二维码

## 4. 用户界面设计

### 4.1 设计风格

**主题色彩**:
- 主色调: 深蓝色系 (#1E3A8A, #3B82F6) - 代表科技与专业
- 辅助色: 青色 (#06B6D4) - 用于强调和交互元素
- 成功色: 绿色 (#10B981) - 表示正常状态
- 警告色: 橙色 (#F59E0B) - 表示需要注意
- 危险色: 红色 (#EF4444) - 表示删除或错误
- 背景: 浅灰色 (#F3F4F6) - 柔和的背景色
- 卡片: 白色 (#FFFFFF) - 内容容器

**字体方案**:
- 标题字体: "思源黑体", "Source Han Sans CN", "Microsoft YaHei", sans-serif - 现代感和可读性
- 正文字体: "思源黑体", "Source Han Sans CN", "Microsoft YaHei", sans-serif - 统一字体系统
- 代码/ID字体: "JetBrains Mono", "Fira Code", "Consolas", monospace - 用于设备ID和MAC地址显示

**按钮样式**:
- 主要按钮: 圆角(8px), 渐变背景, 轻微阴影, 悬停时上浮效果
- 次要按钮: 圆角(8px), 边框样式, 悬停时背景填充
- 危险按钮: 红色背景, 确认弹窗保护

**布局风格**:
- 卡片式布局, 每个设备以卡片形式展示
- 左侧固定导航栏(可选)
- 顶部操作栏(搜索、筛选、添加按钮)
- 响应式网格布局, 不同屏幕尺寸自适应

**图标风格**:
- 使用线性图标风格
- 设备类别配专属图标(传感器用信号图标,核心板用芯片图标等)

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|---------|---------|--------|
| 设备管理页面 | 顶部操作栏 | 搜索框(带图标)、类别下拉筛选器、"添加设备"按钮(主题色) |
| 设备管理页面 | 设备卡片网格 | 3列响应式网格,每个卡片包含:设备图标、ID(等宽字体)、名称、类别标签、状态指示、快捷操作按钮 |
| 设备管理页面 | 空状态提示 | 大型插图+引导文案+"添加第一台设备"按钮 |
| 设备详情页面 | 设备基本信息卡片 | 左侧设备图标、右侧信息列表(标签:值格式),信息项用分隔线区分 |
| 设备详情页面 | 二维码展示卡片 | 居中显示二维码,下方显示设备ID,"下载二维码"按钮 |
| 设备详情页面 | 硬件规格表格 | 表格形式展示核心型号、MAC地址等技术参数 |
| 设备详情页面 | 时间线组件 | 垂直时间线,显示入库时间、修改记录等重要节点 |

### 4.3 响应式设计

**桌面优先设计**:
- 桌面端(≥1280px): 设备卡片网格3列展示,侧边栏固定显示
- 平板端(768px-1279px): 设备卡片网格2列,侧边栏可折叠
- 移动端(<768px): 设备卡片单列,顶部导航栏简化,侧边栏隐藏

**触摸优化**:
- 卡片可触摸滑动查看更多操作
- 按钮最小触摸区域44px×44px
- 表单控件间距适当增加

### 4.4 动画与交互

**页面过渡**:
- 卡片加载: 渐入动画(stagger效果)
- 卡片悬停: 轻微上浮(translateY: -4px)并增加阴影

**微交互**:
- 按钮点击: 缩放效果(scale: 0.95)
- 表单提交: 提交按钮loading状态
- 删除操作: 确认对话框,带淡入淡出动画
- 二维码生成: 加载动画,生成完成后的展示动画

**反馈提示**:
- 成功操作: 绿色Toast提示,自动消失
- 错误提示: 红色Toast提示,需手动关闭
- 表单验证: 字段级实时验证,错误信息显示在字段下方
